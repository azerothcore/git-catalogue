How to add your repository to the catalogue

Quick Start

To add your repository to this catalogue, you need to add specific GitHub topics to your repository. These topics act as tags that help organize repositories into different categories.

Step 1: Add GitHub Topics to Your Repository

What are GitHub topics? Topics are labels that help categorize your repository. You can add them in your repository settings on GitHub.

Global Repository Discovery

This catalogue searches globally across all of GitHub for repositories with the required topics. This means:

  • Your repository can be in any GitHub organization or personal account
  • You don't need to be part of any specific GitHub organization
  • Simply adding the correct topic is enough to get your repository included
  • The catalogue refreshes automatically every 30 minutes

Step 2: Prepare Your Repository

Required Information (add to your README.md):
  • Name: Clear name of your module/script/tool Example: "Player Statistics Module"
  • Description: Brief description that will appear in the catalogue This should be in your repository's description field on GitHub
  • Author: Your username or the original author if this is a fork Important for proper attribution
  • Installation Instructions: How to install and configure your module Make it easy for users to get started
  • License: The license under which you're releasing your code Recommended: GNU Affero General Public License v3.0

Step 3: Enhance Your Repository (Recommended)

A comprehensive README.md file will be displayed on your module's detail page. Include:

  • Detailed description of features
  • Screenshots or examples
  • Installation and configuration steps
  • Requirements and compatibility
  • Troubleshooting section
GitHub README Guide

Add an icon.png file in your repository root to customize your module's appearance.

  • Recommended size: 64x64 pixels or larger
  • Format: PNG with transparent background
  • Keep it simple and recognizable
If no icon is provided, the repository owner's avatar will be used.

  • Configuration options: Provide extensive configuration to avoid code modifications
  • Database patches: Use db_assembler for SQL patches if applicable
  • Code quality: Follow standard coding best practices
  • Documentation: Include inline comments and documentation
  • Version control: Use proper .gitattributes and .gitignore files

Step 4: Submit Your Repository

Processing Time

After adding the required topics to your repository, it may take up to 30 minutes for your repository to appear in the catalogue, as the data is updated periodically.
